    数组的正确打开方式

    🐟 2022/1/29

    # 操作数组的正确姿势

    稀疏数组： 就是存在空位的数组

    • 1.创建数组的方式
    const arr1 = []

    const arr1 =  Array(3) // [empty * 3] 避免使用这样的方式创建数组, 这种数组属于稀疏数组

    const arr1 = Array("3") // ["3"] 避免使用这样的方式创建数组, 这种数组属于稀疏数组
      1. 使用 ES6 新增方法来弥补构造函数的缺陷
    const arr1 = Array.from({length: 3}) // [undefined, undefined, undefined]
      1. 空位的检测 in
      • 空位是没有任何值，可以用 in 运算符检测
    const a = [,,,]; // [empty * 3]
0 in a; // false
const b = [1, 2, 3];
0 in b; // true
      1. 造成稀疏数组的操作

      • delete 操作符

        使用 delete 操作符可以将数组项从数组中删除，数组的长度并不会发生变化，而是会留下一个空位

    const b = [1, 2, 3];
delete b[0];
console.log(b); // [empty, 2, 3];
    • 构造函数
    const a = new Array(3);
console.log(a); // [empty, empty, empty];
    • 在数组字面量中省略值
    [,,,] //[empty * 3]
    • 指定数组索引大于数组长度
    const c = [];
c[10] = 0;
console.log(c); // [empty * 10, 0]
    • 指定数组长度大于当前数组长度
    const a = [];
a.length = 10; // [empty * 10]

    我们在平时要尽量避免创建和使用稀疏数组，因为在 ES6 之前的方法，对稀疏数组的处理存在很多不统一的地方

      1. 操作的不统一

      ES5 对空位的处理，已经很不一致了，大部分情况下会忽略空位 forEach(), filter(), reduce(), every() 和 some() 都会跳过空位. map() 会跳过空位，但会保留这个值 join() 和 tostring() 会将空位视为 undefined，而 undefined 和 null 会被处理成空字符串

      ES6 则是明确将空位转为 undefined Array.from 方法会将数组的空位，转为 undefined 扩展运算符 (...) 也会将空位转为 undefined fill() 会将空位视为正常的数组位置 for...of 循环也会遍历空位 entries()、keys()、values()、find()和 findIndex() 会将空位处理成 undefined

    操作的不统一可能会产生很多意料之外的结果，建议不要创建和使用稀疏数组

      1. 固定长度的 undefined 数组创建
      • Array.apply
    // 将类数组对象转换为数组对象
// 在创建 undefined 值的数组时有些奇怪和繁琐, 但是结果远比 Array(3) 更准确可靠
Array.apply(null, {length: 4});// [undefined, undefined, undefined, undefined]
    • Array.from
    // 将类数组对象转换为数组对象
Array.from({length: 4});// [undefined, undefined, undefined, undefined]
    • 扩展运算符
    [...Array(4)];// [undefined, undefined, undefined, undefined]

    总而言之

    • 不要创建和使用稀疏数组
    • 避免使用 Array 构造函数创建数组
    • 推荐使用对象字面量，of()，from()，扩展运算 ... 创建数组
